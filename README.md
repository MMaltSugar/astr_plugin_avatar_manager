
=======
## astr_plugin_avatar_manager

# Bot 形象管理插件 for AstrBot

版本: 1.0.0 | 作者: MMaltSugar


# 项目简介

费尽心思给Bot整了一套漂亮衣服，bot转头就忘，怎么办？

于是我找来免费的豆包老师，猪脑辅助豆包编程整出了这个bot换装插件。实际上就是个文本管理+插入的功能，改一改或许还可以用来做其他文本管理？

因为是猪脑辅助豆脑写的，所以可能会有稀奇古怪的问题？我猪脑看了一遍应该是没问题的。（其实这个README也基本上是豆包写的）

这是一款为 AstrBot 开发的 Bot 形象管理插件，专为解决多对话场景下 Bot 形象混乱、覆写出错、人设不统一的问题设计。插件采用「指针式引用」的无冗余数据结构，彻底解决形象覆写混乱问题，支持 LLM 全自主管理 Bot 形象，自动将形象设定注入到 LLM 上下文，让 Bot 严格遵守人设形象。

如果你的astrbot有绘图功能，那这个插件应该会很好玩（至少我玩的很爽）


# 核心特性

对话级形象完全隔离：每个私聊 / 群聊对话拥有独立的形象库，不同对话的形象互不干扰，完美适配多场景人设需求

无冗余稳定数据结构：当前形象仅作为「名称指针」引用形象列表，切换 / 覆写零成本，从根源杜绝数据混乱、覆盖错误问题

开箱即用默认形象：新对话无形象时，自动生成「常服 + 居家服」两套预设形象，无需手动配置即可使用

LLM 全自主管理能力：提供完整的工具链，大模型可自主完成形象的创建、切换、修改、删除全流程操作

规范的工具调用规则：明确「1-3 条词条修改用单条修改工具，4 条及以上批量修改用覆写工具」，减少冗余调用，提升执行效率

自动上下文注入：每次 LLM 请求前，自动插入「当前形象详情 + 可用形象列表」到指定位置，确保 Bot 严格遵守形象设定

管理员指令：提供后台管理指令，支持手动配置、管理对话形象


# 安装部署


环境要求

Astr_Bot

Python 3.10+

已正确部署 AstrBot 机器人框架


安装步骤

在 AstrBot 的plugins目录下，新建文件夹astrbot_plugin_avatar_manager

将以下 4 个文件放入该文件夹中：

metadata.yaml：插件元数据配置

_conf_schema.json：插件配置项定义

main.py：插件核心代码

README.md：其实这个可以不放？

重启 AstrBot，插件会自动加载并初始化

可在 AstrBot 的 Web 管理页中，找到「Bot 形象管理」插件，调整相关配置

# 使用指南


管理员指令

所有指令均需管理员权限执行，直接在对话中发送即可生效。


表格

指令	用法示例	说明

查看bot形象	查看bot形象	查看当前对话的 Bot 所有形象数据，包含当前形象详情和完整形象列表

创建bot形象	创建bot形象 泳装 海边度假穿搭 上衣=粉色比基尼 下着=粉色比基尼 发色=金色	创建新的形象，参数格式：形象名 简介 词条1=值1 词条2=值2...，简介可选

切换bot形象	切换bot形象 居家服	将当前对话的 Bot 当前形象切换为指定形象

删除bot形象	删除bot形象 旧泳装	从形象列表中删除指定形象，无法删除当前正在使用的形象

清空当前对话形象	清空当前对话形象	清空当前对话的所有形象数据，不可恢复


# LLM 工具说明

插件为大模型提供了 4 个专用工具，LLM 可根据对话场景自主调用，无需人工干预。


表格

工具名称	核心功能	调用规则

create_avatar_outfit	创建 / 覆盖形象列表中的指定着装，支持自定义词条和简介	修改 4 条及以上形象词条时，必须使用本工具覆写整个形象，禁止逐条修改

select_avatar_outfit	切换当前形象，从形象列表中选择指定着装设为当前使用的形象	仅切换形象引用指针，无需覆写任何数据，零成本切换

modify_avatar_field	修改指定着装的单个词条或简介	仅用于修改 1-3 条形象词条 / 简介，4 条及以上批量修改请使用创建工具

delete_avatar_outfit	从形象列表中删除指定着装	无法删除当前正在使用的形象，需先切换到其他形象后再删除

表格

字段	说明

conversation_id	对话唯一 ID，与文件名对应

current_outfit	当前使用的形象名称，仅作为指针指向outfits中的键，无冗余数据

outfits	形象列表，键为形象名称，值为形象详情（简介 + 词条）

description	单形象的简介，限制 50 字以内

fields	形象词条键值对，支持自定义扩展

# 常见问题 FAQ

Q: 新对话没有形象怎么办？

A: 插件会在新对话首次触发 LLM 请求时，自动创建「常服 + 居家服」两套默认形象，无需手动创建。也可以通过管理员指令创建bot形象手动添加自定义形象。

Q: 切换形象后，Bot 还是不遵守新形象设定？

A: 1. 检查配置项llm_insert_position，建议改为system_prompt_start提高形象设定的优先级；2. 确认对话 ID 正确，不同对话的形象是完全隔离的；3. 重启 AstrBot 重新加载插件。

Q: 升级插件后，之前的形象会丢失吗？

A: 不会。插件内置了旧数据自动迁移功能，会无损将 1.x 版本的形象数据转换为 2.0.0 的新结构，原有形象完全保留。

Q: 怎么限制 LLM 不能随意修改 / 删除形象？

A: 打开main.py，找到__init__方法中的self.context.add_llm_tools(...)代码，删除不需要的工具（如删除DeleteAvatarOutfitTool()即可禁止 LLM 删除形象），然后重启 AstrBot 即可。

Q: 群聊里的形象是所有人共用的吗？

A: 是的。插件以「对话 ID」为隔离单位，每个群聊对应一个独立的对话 ID，群内所有成员共用一套 Bot 形象；私聊对话则是个人独立的形象，互不干扰。

Q: LLM 调用工具时提示形象不存在？

A: 先通过查看bot形象指令确认形象列表中存在该名称的形象，注意形象名称区分中英文、大小写，需完全一致才能正常调用。
